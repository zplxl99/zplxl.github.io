Handler机制之Message

发表于2022-01-05|更新于2022-01-05|FrameworksHandler
|阅读量:

一.概述

  ”消息“是Android消息机制中信息的载体，它包含了在整个消息传递过程中想要传送的数据。所以要学习Handler机制，我们先来看看消息载体Message。

  Message类定义在/frameworks/base/tree/core/java/android/os/Message.java中。我们来看下官方的定义：

  官方的定义很好理解：Messagee是包含了描述信息和数据对象的用于在消息机制中发送给Handler的对象，他包含了2个整型域和1个对象域，通过这些域可以进行信息传递。虽然Message有公有构造函数，但是还是建议使用obtain()方法来获取Message对象，这种创建方式会重复利用缓存池中的对象而不是直接创建新对象。

二.源码解读

1.成员变量

  • int what ：这个整型对象可以理解成消息的命名空间。用户定义的消息代码，以便收件者可以识别这条消息是关于什么的。每个Handler都有自己的消息代码命名空间，因此不用担心自己的名称空间与其他处理程序发生冲突；
  • int arg1：如果只需要存储几个整型数据，arg1和arg2是setData()的低成本替代品；
  • int arg2：如果只需要存储几个整型数据，arg1和arg2是setData()的低成本替代品；
  • Object obj：发送给接收器的任意对象，当使用Message对象在线程间传递消息时，如果它包含一个Parcelable的结构类，此字段必须为非空。其他的数据传送则使用setData()；
  • Messenger replyTo：可选的Messenger对象，具体如何使用取决于发送者和接受者；

2.obtain方法

  Message类中有多个obtain方法：

  • obtain()
  • obtain(Message orig)
  • obtain(Handler h)
  • obtain(Handler h, Runnable callback)
  • obtain(Handler h, int what)
  • obtain(Handler h, int what, Object obj)
  • obtain(Handler h, int what, int arg1, int arg2)
  • obtain(Handler h, int what,int arg1, int arg2, Object obj)

  可以看到，会先将Message中的各个成员变量赋值初始值，然后判断当前缓冲池的大小是否超过MAX_POOL_SIZE（MAX_POOL_SIZE定义为50），如果没有超过的话就将当前的Message放入到next中，同时将缓冲池的size值加一。

三.消息缓冲池

  在obtain和recycle方法的实现中都可以看到sPoolSize的加一/减一操作，都依赖缓冲池。我们看下缓冲池的具体实现：

1.获取（obtain）

  • 第一步：判断sPool是否为空，如果缓冲池为空，则直接new一个Message返回；
  • 第二步：Message m = sPool；将缓冲池中的消息取出，赋值给m；
  • 第三步：sPool = m.next；将缓冲池中的下一个可复用的Message对象（m.next）赋值为缓冲池中的当前对象。（如果消息缓冲池之前就一个，则此时sPool=null）；
  • 第四步：m.next = null；将m.next置为Null，因为之前对象已经取出来了，所以无所谓；
  • 第五步：m.flags = 0 ；设置m的标志位，标志位正在被使用；
  • 第六步：sPoolSize– ；这时候Message对象已经被取出来了，因此缓冲池的size减一；

2.回收（recycle）

  若当前缓冲池不为空，当前回收的消息是Message B，前一个被回收的对象是Message A

  • 第一步：next = sPool ；此时sPool是Message A，同时赋值给next，next也是Message A；
  • 第二步：sPool = this；this是当前的Message B，将他赋值给sPool，这时候sPool也是Message B；
  • 第三步：sPoolSize++；缓冲池size加一，一直到MAX_POOL_SIZE（50）为止；

  如果当前缓冲池为空

  • 第一步：next = sPool ；缓冲池为空，那么此时sPool是Null，赋值给next，此时next也是Null；
  • 第二步：sPool = this；将当前这个message作为消息缓冲池中下一个被复用的对象；
  • 第三步：sPoolSize++；默认为0，此时为1，将消息缓冲池的数量+1，这个变量依然是全系统共享的；

3.示意图

  从获取和回收的过程来看，消息缓冲池就是一个链表，不管是获取还是回收都要保证线程同步的，这样就会始终形成一条链式结构，通过sPool指针的反复+/- 来实现消息对象的获取和回收。
