渲染性能

发表于2021-08-28|更新于2021-08-28|性能优化基础知识
|阅读量:

渲染性能是你在应用打造过程中最常见的性能问题。

一.概述

  一方面，你的设计师想为用户呈现最棒的使用体验；但是另一方面，这些华丽的图形和转场并不是 在每一台设备上都能很好地运行。 系统会试图每16毫秒就对你的活动进行重新绘制，这意味着你的应用需要在16毫秒内运行所有的相 关逻辑来更新屏幕，从而达到60帧每秒的效果。如下图：

  如果你错过了这个时间，比如花了24毫秒，你就会发现丢帧的情况。系统试着刷新一个新的图像， 但是它还没准备好。所以什么都刷新不出来。用户会在32毫秒而不是16毫秒内看到同一帧画面。如下图：

  这种丢帧的现象就是用户所说的延迟或卡顿。Lag/Jank/Slow/Hitching…丢帧期间的所有动画会成为 运行过程中的卡顿，很容易被发现。当与系统互动的时候，这个情况会更加严重，比如拖动ListView或 者输入数据时。

二.丢帧的原因

  错过这个16毫秒的原因有很多。例如，

  • 可能花了过多的时间重新绘制你的视图层级的某些部分，这会浪费CPU周期：

  • 绘制了太多的对象，重复叠加，这也会浪费宝贵的时间进行像素着色：

  • 一次次的运行大量动画，这会导致你的CPU和GPU组件上大量的运行：

三.常用工具

  • Hierarchy Viewer

  用来理解复杂的活动，这可以查出确认时间过长、浪费时间太多重新绘制等问题。

  • Profile GPU Rendering/Show GPU Overdraw/GPU View Updates

  这些都是开发者选项中的On-Device工具，这些都可以帮助检测出应用出现的问题。

  • Traceview

  对于出现在CPU代码中的棘手的渲染漏洞，Traceview是很适合的工具来检测到底是什么问题导致超 过了16毫秒每帧的界限。
